НАЗНАЧЕНИЕ - редактирование блок-схем программ.
             конвертация из кода программы в блок-схему.

ОСНОВНАЯ ИДЕЯ  -  изобразить блок-схему с произвольным по 
величине (имеется в виду текст)  содержанием.  В большинстве
редакторов, если  и имеется возможность автоматически менять
размер компонентов, то все стрелки и переходы все равно надо
расставлять вручную.  В FCEditor это делается автоматически.

ВНИМАТЕЛЬНО прочитайте раздел про импорт - это важно!

//--------------------------------------------------------//
//-----------------Преимущества FCEditor:-----------------//
//--------------------------------------------------------//

 * ИМПОРТ из кода языка Паскаль в блок-схему
 * АВТО-определение размеров и расположения  всех элементов
 * АВТО-конвертация в программный код который демонстрируется 
 * возможность вставки только стандартных элементов алгоритмов 
 * многоязычный интерфейс (русский, английский и др.)
 * возможность разбиения длинных схем на страницы. 

Принцип работы программы - работа с элементами как с объектами.
При наведении на область элемента можно изменить его содержимое 
(если находимся в режиме "Изменение")  либо  при  наведении на 
исходящую из элемента стрелку указателя мыши можно  произвести 
вставку новых элементов (выбранных с помощью переключателей на
панели  инструментов либо из контекстного меню).  Переключится 
в один  из режимов работы можно как из панели инструментов,так
 и из контекстного меню. В строке ввода можно менять текстовое
содержимое элемента.  При установке флажка  -  "автоматическое
обновление" размеры компонента будут подгоняться под его текст
(только нужно указать что текущий режим - редактирование).

// -- Копирование\Вставка\Вырезание блоков --

Копирование\вырезание блока в буфер обмена:
  нужно выделить сам блок, а затем в контекстном меню выбрать
  соответственный пункт.

Вставка блока из буфер обмена:
  нужно навести указатель мыши на стрелку и в контекстном меню
  выбрать пункт "Вставить Блок".

// -- Нумерация блоков -- (пока отключено!)

программа может указывать номера блоков (чтобы в описании можно
было на них ссылаться). Для этого нужно выставить в опциях 
переключатель "Нумеровать блоки".
номера жестко заданы и определяются порядком добавления блоков
в схему.

// -- Разбиение на страницы --

Если ваша схема получается слишком длинная и не размещается на
одной странице, то можно это обойти, если вставить блок
"разбиение" из двух кружков с номерами (как в ГОСТЕ). При
экспорте такой схемы в картинку, файл будет поделен на несколько 
частей (если было два блока "разбиение", то будет три файла):
  fileNN_part01.bmp
  fileNN_part02.bmp
  fileNN_part03.bmp
остается только вставить их, например, в Word.
Примечание. блок "разбиение" можно вставить только в основную
последовательность блоков. Номера жестко заданы и определяются
порядком добавления блоков "разбиение" в схему.

кнопки панели инструментов

+ Добавить - режим добавления нового элемента.  Чтобы добавить 
  новый элемент надо, дважды щелкнуть указателем мыши по стрелке,
  где предполагается добавить новый элемент. 
Х Удалить -  режим удаления элементов.Дважды щелкаем по элементу
  и он удалится.
Т Изменить - режим изменения содержания элемента. дважды щелкаем
  по элементу, при этом его содержание становиться доступным для
  редактирвания и его можно менять.  Если при этом установить
  флажок "Обновлять при редактировании",то сразу будет происходит
  перерисовка схемы.
Н Передвинуть  -  в этом режиме схему можно перемещать удерживая
  левую клавишу мыши. Остался как атавизм. Сейчас схему можно 
  двигать любым инструментом.
- В начало - перемещение в начало координат.

//--------------------------------------------------------//
//--------------Замечания об импорте из кода--------------//
//--------------------------------------------------------//

  ПРО С++

* с версии 1.6 есть ИМПОРТ из С++
  к сожалению пока нужно вручную удалять команды препроцессора
* блок иф пока должен иметь ОБЕ ветки!

  ПРО Паскаль

* пока распознаються файлы программ. Они должны начинаться с
  заголовка program NameProg; (не забыли еще синтаксис?)
  пока есть проблемы с распознаванием составных имен функций и
  процедур, поэтому вместо 
  program pr1;
  procedure Class1.method1;
  begin
    //...
  end;
  begin
   //...
  end.
  на время импорта замените точку на латинский символ или _
  вот так, например
  //...
  procedure Class1_method1;
  //... 

  а уж если приперло, то после импорта замените в обратку.

* исходик должен быть синтаксически правильным

* если файл был создан в DOS-кодировке с пояснениями на русском 
  - откройте его в например, в Word и сохраните как текст 
  с кодировкой Windows.
  Это нужно если в тексте программы есть русские символы.
 
* конвертер импорта НЕ распознает блока case ... of ... end; 
  более того, при его наличии почти гарантировано импорт завершиться
  с ошибкой, .. единственно, что могу рекомендовать
  это заменять кейс вручную на структуру if .. else ..
  например оператор:
    case numer of
      1: DoSomething1;
      2: DoSomething2;
      3: DoSomething3;
    end;
  можно заменить на:
    if numer = 1 then
      DoSomething1
    else
      if numer = 2 then
        DoSomething1
      else
        if numer = 3 then
          DoSomething1;
  
  в ближайших планах 
    во-первых чтение кейса в иф автоматически
  а во-вторых дальнейшем и рисование этого оператора.
